Sentiment analysis can be a helpful tool in analyzing consumer behavior. It can especially helpful when a language model can predict consumers’ sentiments on products based on social media postings. In this project, we used fashion and beauty dataset from Kaggle (https://www.kaggle.com/competitions/fashion-and-beauty-reviews) to build a stepping stone for a model that can analyze consumers’ feelings about particular products based on their social media postings.

DATA PROCESSING

The training data consists of 1,003,684 amazon reviews and ratings. The data is very imbalanced with 5-star reviews overrepresentation in the training data. This is shown in Figure 1.

Leaderboard Probing

We used a series of test submissions to check that this imbalance occurs in the test. The first submission of 2.5 for each review resulted in a public leaderboard score of 2.032. Changing the constant value to the train data review mean of 3.967 resulted in a public leaderboard score of 1.399. Leaderboard probing may indicate that the imbalanced distribution seen in the train data matches the test set distribution.

Modeling methodology

Universal Language Model Fine-tuning Method

For better understanding of ULMFiT, visit https://docs.fast.ai/tutorial.text.html.

Following code shows how to prepare data to train the language model.

# Import libraries
from fastai.basics import *
from fastai.callback.all import *
from fastai.text.all import *

#Get path for data
path = Path('/kaggle/input/fashion-and-beauty-review/')

# Read in data
df_train = pd.read_csv(path/'review_train.tsv', sep='\t')
df_test = pd.read_csv(path/'review_test_post.tsv', sep='\t')

# Change ratings from string to float for regression
df_train.loc[:, 'rating'] = df_train.loc[:, 'rating'].astype(float)

#Drop unnecessary columns in test data
df_test.drop(['summary', 'product'], axis=1, inplace=True)

# Get random 50000 rows
ids = np.random.randint(0, len(df_train), 50000)

# Create DataLoaders
dls = TextDataLoaders.from_df(df_test, is_lm=True, valid_pct=.1, text_col='review')

# Make a language model with AWD_LSTM architecture and wikipedia pre-trained model with mixed precision for faster training
learn = language_model_learner(dls, AWD_LSTM, metrics=[accuracy], path='/kaggle/working', wd=0.1).to_fp16()

# Train head
learn.fit_one_cycle(1, 1e-2)

# fine tune the entire model
learn.unfreeze()
learn.fit_one_cycle(10, 1e-3)

# Save the weights
learn.save_encoder('finetuned_on_test')

Large Ensemble Model Method

To speed up the model development and training we use just the first 50,000 rows of the training data for this step.

Single model training may lead to strong predictors, however this often comes at the expense of computation time. In addition, many weak performers ensembled may outperform and generalize better than single-model models.

To take advantage of pre-trained language models we identified and downloaded 19 pre-trained models trained on available open datasets. These models included sentiment analysis models, emotion detection models, and review rating models. Each model was used to provide inference on the train and test sets. To evaluate each model we trained a linear model on the outputs of each model and evaluate with a 90-10 split. The code and results are shown in the code and table below.

Results and Analysis

The figure below shows the histogram of model outputs by class label. This plot demonstrates the difficulty of predicting between similar ratings for low rated reviews. Significant overlap is visible for 1-star and 2-star reviews. This is reflected also in the RMSE scores for each category where the 1-star and 2-star classes have the highest prediction error.

The confusion matrix for the linear model predictions reveals the same effect occuring for high rated reviews as well with significant errors when predicting reviews with 4-stars and 5-stars.

Future Work

Future work on this project could focus on increasing the training set size. We were forced to use a subset of the training data to speed up model development times, this limit could be removed to help increase model performance.
